## Cross-Phase Integration

**Integration checker result: 11/11 exports wired, 0 orphaned, 0 broken flows, 6/6 E2E flows complete.**

### Critical Seam: Phase 14 → 15 Catalog-Known Invariant — SOUND

Phase 15's `MetadataService._engine_catalog()` and the resolver design explicitly
rely on Phase 14's IDENT-01 guarantee (catalog rejected at connect time). The chain
holds end-to-end:
1. `DatabricksDialect.create_engine` raises `ValueError` on empty/None catalog.
2. Both connect paths route it through `_require_databricks_catalog` → `ConnectionError` with `SHOW CATALOGS` listing.
3. `list_schemas` reads `engine.url.query["catalog"]` directly (no silent fallback).
4. `resolve_identifier` returns `catalog=None` safely when no override given — the engine is already catalog-scoped.

### Resolver Wiring — CONSISTENT across all 7 namespace-aware tools

`resolve_identifier`: get_table_schema, get_sample_data, get_column_info,
find_pk_candidates, find_fk_candidates. `_assert_catalog_allowed`: list_schemas,
list_tables. `connect_database`/`execute_query` correctly excluded.

### Integration WARNING (non-blocking)

`connect_with_url` with a structurally invalid Databricks URL (missing host) re-raises
the raw `_kwargs_from_url` ValueError instead of the enriched catalog error. Only the
error-message quality degrades on an unusual input; the catalog-missing path (the
IDENT-01 target scenario) works correctly. Tracked as tech debt, not a gap.
