## Trust Boundary

MCP tool args (`catalog` / `schema` / `table` / `column`) flow from an untrusted client
through `resolve_identifier` (explicitly NOT a SQL sanitizer) into raw, non-parameterized
Databricks SQL (`DESCRIBE TABLE`, `DESCRIBE EXTENDED`, `SHOW TABLES IN`, `information_schema`,
and TSQL-authored aggregate SQL transpiled via sqlglot). The injection defense is therefore
per-segment identifier quoting, not bound parameters.

   reflection and statistics run as the connected Databricks principal; Unity Catalog ACLs
   enforce read authorization server-side. No new exposure beyond existing

   resolved catalog (`SHOW TABLES IN catalog.schema`), not across arbitrary catalogs; UC ACLs
   enforce. Accepted.

   connected principal; no new exposure beyond `get_column_info`'s existing cross-catalog
   `table_exists`. Accepted.
4. **T-15.1-15 — Live cross-catalog read of cerner_src.** Read-only metadata/statistics as
   connected principal; no write, no new credential; UC ACLs enforce. Verified live in UAT.
   Accepted.
5. **T-15.1-SC — Supply chain.** No new dependencies; all libraries are pre-existing pinned
   deps. Accepted.

## SQLi-Class Control Fixes (load-bearing for T-15.1-01/03/06/09/10)

- **CR-01 (commit 537680c) — VERIFIED PRESENT.** `DatabricksDialect.quote_identifier` now
  doubles embedded backticks: `f"\`{identifier.replace('\`', '\`\`')}\`"`
  (`src/db/dialects/databricks.py:131`), mirroring `MSSQLDialect.quote_identifier`'s
  `]`→`]]` (`src/db/dialects/mssql.py:110`). This is the defense for the backtick reflection
  paths (reflect_columns, list_tables, DESCRIBE EXTENDED).
- **WR-04 (commit b49d525) — VERIFIED PRESENT.** Shared `quote_tsql_identifier()` helper
  (`src/analysis/_sql.py:14-32`) doubles `]`→`]]`. No raw `[{...}]` identifier interpolation
  remains in the analysis files — the only `[{...}]` occurrence is the helper's own definition.
  All catalog/schema/table/column TSQL-bracket segments in pk_discovery.py, fk_candidates.py,
  and column_stats.py route through it.

Test corroboration: 44 catalog/injection/no-USE-CATALOG/MSSQL-gate tests pass, 50 skipped
(dialect-marker opt-outs), 0 failures.
