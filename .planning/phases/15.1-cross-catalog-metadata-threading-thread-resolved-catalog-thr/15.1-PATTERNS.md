### `src/analysis/pk_discovery.py` (service, reflection)

**Analog:** `src/db/metadata.py::_get_databricks_columns` (raw-SQL bypass) + the in-file `_qualified_table` field.

**Current constructor** (lines 39-52) — has NO catalog arg, builds 2-part only:
```python
def __init__(
    self,
    connection: Connection,
    schema_name: str,
    table_name: str,
    dialect: "DialectStrategy | None" = None,
    inspector: "Inspector | None" = None,
):
    self.connection = connection
    self.schema_name = schema_name
    self.table_name = table_name
    self._dialect = dialect
    self._inspector = inspector
    self._qualified_table = f"[{schema_name}].[{table_name}]"   # 2-part TSQL brackets
```

**Change — add `catalog` arg + 3-part TSQL `_qualified_table`** (RESEARCH §Code Examples; Pitfall 4 — keep it TSQL brackets, NOT backticks, so `transpile_query` handles it):
```python
def __init__(self, connection, schema_name, table_name,
             dialect=None, inspector=None, catalog=None):   # NEW: catalog
    ...
    self._catalog = catalog
    if catalog:
        self._qualified_table = f"[{catalog}].[{schema_name}].[{table_name}]"
    else:
        self._qualified_table = f"[{schema_name}].[{table_name}]"
```
`_column_is_unique` (lines 290-310) already interpolates `self._qualified_table` and runs it through `transpile_query(uniq_sql, self._dialect)` — so the structural/uniqueness path is fixed by the constructor change alone. **Verified clean transpile** (RESEARCH line 201): `[cat].[sch].[tbl]` → `` `cat`.`sch`.`tbl` ``.

**Re-route the Inspector-backed reads on the cross-catalog branch.** These three call sites are catalog-blind (they call `self._inspector.get_*` which binds `self.catalog`):
- `_get_constraint_candidates_inspector` lines 168-176: `self._inspector.get_columns(...)`, `self._inspector.get_pk_constraint(...)`
- line 193: `self._inspector.get_unique_constraints(...)`
- `_list_all_columns` lines 265-272: `self._inspector.get_columns(...)`

When `self._catalog` is set AND dialect is Databricks, replace `get_columns` with the DESCRIBE-TABLE raw-SQL pattern below. Constraints (PK/UNIQUE) come from `information_schema.table_constraints` / `key_column_usage` qualified to the catalog (RESEARCH §Architecture row 54), since Databricks Inspector PK/FK are also `self.catalog`-bound.

**Reference raw-SQL pattern to copy** — `src/db/metadata.py::_get_databricks_columns` lines 966-996:
```python
quoted_catalog = self._dialect.quote_identifier(catalog)
quoted_schema = self._dialect.quote_identifier(schema_name)
quoted_table = self._dialect.quote_identifier(table_name)
qualified = f"{quoted_catalog}.{quoted_schema}.{quoted_table}"
with self.engine.connect() as conn:
    result = conn.execute(text(f"DESCRIBE TABLE {qualified}"))
    rows = result.fetchall()
# parse rows[i][0]=col_name, rows[i][1]=data_type; STOP at blank or col_name.startswith("#")
```
> Note the analysis classes hold a live `self.connection` (not `self.engine`) — use `self.connection.execute(text(...))` directly; do NOT open a new `self.engine.connect()`. Every segment quoted via `self._dialect.quote_identifier` (injection defense T-mwr-02; DESCRIBE does not support bound params — RESEARCH §Security).

For the analysis classes, prefer the real `dialect` fixture from `tests/conftest.py` lines 49-69 (`@pytest.mark.dialects('databricks')` gives a real `DialectStrategy` whose `quote_identifier` produces real backticks) over the local `_make_databricks_dialect` MagicMock.

**Why existing analysis-test mocks DON'T catch CR-02 (Pitfall 2):** `tests/unit/test_pk_discovery.py` lines 40-48 use an **order-dependent `side_effect` LIST** that returns rows regardless of which catalog the SQL targets:
```python
conn = MagicMock()
conn.execute.side_effect = [_mock_result(pk_rows), _mock_result(uq_rows)]   # catalog-BLIND
```
A passing test here proves nothing about catalog targeting.

**Required upgrade — catalog-discriminating fake** (RESEARCH §Mock strategy lines 436-445). Replace the list with a content-inspecting function that returns rows ONLY when the executed SQL carries the expected 3-part name:
```python
EXPECTED = "`bmtct`.`dbo`.`orders`"   # or the TSQL pre-transpile form [bmtct].[dbo].[orders]

def discriminating_execute(stmt, *args, **kwargs):
    sql = str(stmt)
    assert "USE CATALOG" not in sql          # Pitfall 1 guard (statelessness)
    if EXPECTED in sql or "[bmtct].[dbo].[orders]" in sql:
        return _mock_result(EXPECTED_ROWS)   # right catalog -> data
    return _mock_result([])                  # wrong/default catalog -> empty (would be a false negative)

conn.execute.side_effect = discriminating_execute
# assert candidates non-empty -> proves the statement carried the catalog
```
Plus a pure-sqlglot assertion (no DB) that the 3-part `_qualified_table` transpiles correctly:
```python
import sqlglot
assert sqlglot.transpile("SELECT 1 FROM [cat].[sch].[tbl]", read="tsql",
                         write="databricks")[0] == "SELECT 1 FROM `cat`.`sch`.`tbl`"
```

## Shared Patterns

### Catalog-aware raw-SQL bypass (the core pattern, applied to all 3 analysis classes)
**Source:** `src/db/metadata.py::_get_databricks_columns` lines 966-996 (DESCRIBE TABLE), `_list_tables_databricks` lines 352-362 (SHOW TABLES IN), `table_exists` lines 1142-1157.
**Apply to:** Every Inspector-backed read in `pk_discovery.py`, `fk_candidates.py`, `column_stats.py`, on the cross-catalog branch ONLY (`self._catalog and dialect.name == "databricks"`). Default-catalog path stays on the Inspector (no behavior drift — regression-protected by existing tests).
```python
quoted_catalog = self._dialect.quote_identifier(catalog)
quoted_schema  = self._dialect.quote_identifier(schema_name)
quoted_table   = self._dialect.quote_identifier(table_name)
qualified = f"{quoted_catalog}.{quoted_schema}.{quoted_table}"
result = self.connection.execute(text(f"DESCRIBE TABLE {qualified}"))
```
RESEARCH Open Q2 recommends EXTRACTING this into a shared helper (4th occurrence = Rule of Three crossed) rather than a 4th copy — e.g. a small function in `src/analysis/_sql.py` or `src/db/metadata.py`. Planner decides extract-vs-inline.

### 3-part structural SQL via TSQL brackets + sqlglot (the "easy half")
**Source:** `src/analysis/_sql.py::transpile_query` lines 11-28; canonical 3-part build in `src/db/query.py::get_sample_data` lines 123-139.
**Apply to:** `_qualified_table` in `pk_discovery.py` (line 52) and `column_stats.py` (line 81). Build as `[catalog].[schema].[table]` (TSQL brackets, NOT backticks — Pitfall 4); let `transpile_query` emit `` `cat`.`sch`.`tbl` ``. Verified clean.

### Identifier quoting (injection defense)
**Source:** `dialect.quote_identifier` — `src/db/dialects/databricks.py` lines 124-126 (`return f"`{identifier}`"`).
**Apply to:** Every segment of every raw-SQL 3-part name AND the DESCRIBE EXTENDED fast-path in `column_stats.py` line 367 (T-mwr-02 / T-11-04; DESCRIBE has no bound-param support).

### Statelessness guard (Pitfall 1)
**Apply to:** All cross-catalog SQL. NEVER emit `USE CATALOG` (leaks across the QueuePool, `src/db/connection.py:127,206-216`). Use stateless 3-part names only. Add a test assertion `assert "USE CATALOG" not in sql` on the cross-catalog path.
