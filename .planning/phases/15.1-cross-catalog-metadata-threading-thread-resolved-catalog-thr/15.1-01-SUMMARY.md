# Phase 15.1 Plan 01: CatalogAwareReflector Summary

Extracted the duplicated catalog-aware raw-SQL reflection pattern into a single injection-safe, stateless `CatalogAwareReflector` in `src/analysis/_sql.py` that operates over a caller-owned live connection — the shared contract Wave 2 analysis plans consume (IDENT-08).

## What Was Built

- **`CatalogAwareReflector`** (`src/analysis/_sql.py`): a stateless reflector bound to a live `Connection` + `DialectStrategy`.
  - `reflect_columns(catalog, schema, table) -> list[dict]`: builds a 3-part backtick-quoted name via per-segment `dialect.quote_identifier`, runs `DESCRIBE TABLE`, parses `col=row[0]`/`type=row[1]`, stops at the first blank column name or `#`-prefixed section marker (faithful to `MetadataService._get_databricks_columns`).
  - `list_tables(catalog, schema) -> list[str]`: builds a 2-part backtick-quoted name, runs `SHOW TABLES IN`, extracts `row[1]` (tableName) per row (faithful to `MetadataService._list_tables_databricks`).
- **`tests/unit/test_analysis_sql.py`**: 8 catalog-discriminating tests (databricks-only via `@pytest.mark.dialects('databricks')`), using the real `dialect` fixture so `quote_identifier` emits genuine backticks, with a fake connection whose `execute.side_effect` returns rows only when the expected 3-part name is present in the SQL.

## Threat Mitigations Applied

- **T-15.1-01 (SQLi):** every catalog/schema/table segment is quoted via `dialect.quote_identifier` before concatenation. `test_identifiers_are_quoted_not_raw` asserts a dotted schema appears as a single backtick-quoted token (`` `evil.schema` ``) and never as a raw dotted segment.
- **T-15.1-02 (session-catalog tampering):** reflector emits only fully-qualified names and never mutates the active catalog. `test_never_emits_use_catalog` asserts no executed SQL contains `USE CATALOG`; the source grep gate confirms 0 occurrences.
