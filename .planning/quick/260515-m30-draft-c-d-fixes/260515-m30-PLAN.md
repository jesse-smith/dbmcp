# Quick Task 260515-m30: Draft C/D fixes

## Goal

Fix Defect D (Databricks connection_id collision across `?catalog=` values) so that the engine pool key disambiguates by URL query params. Defect C (URL `?catalog=` ignored on second connect to same host) is downstream of D and resolves automatically.

Defects A and B are out of scope:
- **A** is a separate independent defect at `src/config.py:77` (`catalog: str = "main"`) and `:243` — separate PR.
- **B** is already fixed on the dev branch by commit `6cfe60c`; the prior repro symptom came from probing the frozen `dbmcp` user-level install rather than `dbmcp-test`.

## Approach

**TDD red → green → refactor.** Failing tests first, single-function fix, full pytest + ruff before commit.

The fix is a four-line edit to `_generate_url_connection_id` that folds `parsed.query` (a `dict[str, str]`) into the hashed safe-key string in sorted order. Sorting guarantees `?catalog=a&schema=x` and `?schema=x&catalog=a` produce the same ID. Credentials remain excluded — they live on `URL.username`/`URL.password`, not in `URL.query`.

Considered but rejected: a per-dialect `pool_key_components(url)` interface. Cleaner long-term but adds per-dialect surface and per-dialect tests for a defect with one offender. KISS + reversibility says start with the simpler form; revisit only if a dialect ever needs to *exclude* a query param from the key.

### Task 2 — Implement the fix

**Files:** `src/db/connection.py`

**Action:** Replace the body of `_generate_url_connection_id` (lines 636-643) with:

```python
def _generate_url_connection_id(self, sqlalchemy_url: str) -> str:
    """Generate a deterministic connection ID from a SQLAlchemy URL.

    Credentials are excluded — SQLAlchemy URL.query contains only non-credential
    query params (username/password live on URL.username/URL.password). Including
    sorted query params in the key disambiguates pool entries that differ only
    in dialect-significant params like Databricks ?catalog= and ?http_path=.
    """
    parsed = make_url(sqlalchemy_url)
    sorted_query = "&".join(f"{k}={v}" for k, v in sorted(parsed.query.items()))
    safe_key = (
        f"{parsed.get_backend_name()}://"
        f"{parsed.host or ''}:{parsed.port or 0}"
        f"/{parsed.database or ''}"
        f"?{sorted_query}"
    )
    return hashlib.sha256(safe_key.encode()).hexdigest()[:CONNECTION_ID_LENGTH]
```
