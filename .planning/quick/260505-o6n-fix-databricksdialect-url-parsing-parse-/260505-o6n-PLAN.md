def create_engine(self, **kwargs) -> Engine:
    # Expects: host (required), http_path (required)

SQLAlchemy URL parser to use (same import as MSSQL):

```python
from sqlalchemy.engine.url import make_url
url = make_url(sqlalchemy_url)
# url.host, url.port, url.database, url.username, url.password, url.query (immutabledict)
```

Standard Databricks URL form (produced by the engine the dialect itself builds in kwargs mode):

```
databricks://token:<TOKEN>@<HOST>?http_path=<HTTP_PATH>&catalog=<CATALOG>&schema=<SCHEMA>
```

With a port/database form the user may also supply:

```
databricks://token:<TOKEN>@<HOST>:443/<SCHEMA>?http_path=<HTTP_PATH>&catalog=<CATALOG>
```

URL → kwargs mapping:
- host         <- url.host                                  (required; ValueError if missing)
- http_path    <- url.query.get("http_path")                (required; ValueError if missing)
- token        <- url.password (the username is literally "token"); fallback "" if absent
- catalog      <- url.query.get("catalog", "main")
- schema       <- url.database or url.query.get("schema", "default")
                  (prefer url.database when present — it carries the schema in the path form;
                   fall back to ?schema= query param; default "default")

MSSQL pattern to mirror (already shipped — see src/db/dialects/mssql.py:296 `_kwargs_from_url`):
- Static helper `_kwargs_from_url(sqlalchemy_url, original_kwargs) -> dict`
- URL wins for connection-identity kwargs
- Preserved runtime kwargs set: {query_timeout, pool_config, connection_id,
  disconnect_callback, connection_timeout}
- Conflicting kwargs are logged at debug level, not errored
