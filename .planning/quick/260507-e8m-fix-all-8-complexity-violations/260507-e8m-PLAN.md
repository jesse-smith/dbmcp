# Fix all 8 complexity violations flagged by CI complexipy gate

## Context

`scripts/check_complexity.py` (CI) runs complexipy with `MAX_COMPLEXITY = 15`.
The v2.0 merge landed 8 functions above the limit, so CI is red on `main`.
All 8 are pre-existing functions — no new regressions — but the gate now
sees them on `push: [main]`.

All fixes are behavior-preserving extract-method refactors. The existing
test suite (872 tests, 78 skipped) is the safety net. After each fix,
`uv run python scripts/check_complexity.py` must show the target function
at ≤15. After all 8, the full script must exit 0.

## Must-haves

- `uv run python scripts/check_complexity.py` exits 0 (all 8 pass).
- `uv run pytest -m "not integration and not slow"` passes with no new failures.
- `uv run ruff check src/` passes (no new warnings).
- No behavior change: public signatures and return shapes unchanged for
  all 8 functions.
- One atomic commit per fix (8 commits), each scoped to one function
  plus any new private helpers that support it.

## Tasks

### Task 1 — metadata.py::_parse_databricks_table_properties (26 → ≤15)

**Files:** `src/db/metadata.py`

**Action:** Extract two private helpers:
- `_classify_dte_row(col_name: str, data_type: str, in_partition, in_detail) -> tuple[str | None, bool, bool]`
  returning `(section_change, next_in_partition, next_in_detail)` where
  `section_change` is one of `"partition"`, `"detail"`, `"skip"`, `"data"`, `"reset_partition"`.
- `_apply_dte_detail_row(props: dict, col_name: str, data_type: str) -> None`
  updates `props` based on the Databricks `DESCRIBE EXTENDED` key map.

Main function becomes: quote identifiers, execute, iterate rows, dispatch
to classifier then apply accordingly. Keep the `try/except` error indicator
behavior identical.

**Verify:** Run full test suite focused on Databricks metadata tests:
`uv run pytest tests/ -k "databricks and metadata or parse_databricks" -v`.
Then complexipy check for this function.

**Done:**
- Function body ≤15 cognitive complexity.
- All databricks metadata tests pass.
- `{"_describe_extended_error": ...}` shape on exception preserved.
