# Quick Task 260507-h6j: Move all optional extras to hard dependencies

## Motivation

Every dialect module imports its driver unconditionally at module load time (e.g. `src/db/dialects/azure_auth.py` imports `azure.core`; `mssql.py` imports `azure_auth` which fails if `azure` isn't installed). The `[project.optional-dependencies]` block has been organizational — not a real optionality contract. A fresh `uv tool install dbmcp` without `[all]` crashes on import before any tool runs. Current userbase needs all drivers; the extras split adds failure modes without any payoff.

## Changes

1. **`pyproject.toml`**
   - Move `pyodbc`, `azure-identity`, `databricks-sqlalchemy`, `databricks-sql-connector`, `jupyter`, `notebook` into `[project].dependencies`.
   - Remove `[project.optional-dependencies]` section entirely.
   - Remove `"dbmcp[all]"` from `[dependency-groups].dev` (redundant — all drivers now in main deps).

2. **`tests/unit/test_pyproject_extras.py`**
   - Rewrite: tests previously asserted extras structure; now assert all driver packages are in `[project].dependencies` and the `[project.optional-dependencies]` block is absent.

3. **`src/db/dialects/mssql.py`** (line ~186)
   - Update ImportError string: drop `dbmcp[mssql]` extras syntax. New wording: `"MSSQL support requires pyodbc. Install/reinstall dbmcp to pull it in."` The lazy-import path stays — it's cheap defensive code, just no longer references a non-existent extra.

4. **`src/db/dialects/databricks.py`** (line ~224)
   - Same treatment: drop `dbmcp[databricks]` from the install hint.

5. **`src/mcp_server/_errors.py`** docstring (line ~11)
   - Update the example reference to drop extras syntax.

6. **`tests/mcp_server/test_import_error_messaging.py`**
   - Update fixture strings to match new error wording (fixtures are illustrative — not behavior under test, but keep docs coherent).

7. **README** — no changes needed (no references to extras found in README.md).
8. **CI** — no changes needed (`uv sync --group dev` pulls main deps automatically).

## Verification

- `uv sync --group dev` succeeds.
- `uv run pytest tests/` passes (targeted: `tests/unit/test_pyproject_extras.py` and `tests/mcp_server/test_import_error_messaging.py`).
- `uv run ruff check src/` clean.
- `uv build` succeeds and produced wheel's `METADATA` lists all drivers as hard `Requires-Dist`.

## Out of scope

- Removing the defensive lazy-import guards in dialect modules (they're harmless now; removing them is a separate refactor).
- Updating historical `.planning/milestones/` and `.planning/research/` docs referencing old extras (frozen artifacts).
