<objective>
Replace regex-based _sanitize_identifier() with metadata-based validation that checks user-supplied identifiers against actual database objects, with fail-open fallback on metadata errors.

Purpose: SEC-01 requires identifier sanitization to validate against sys.columns metadata, not regex patterns. This closes the gap where regex-valid-but-malicious identifiers could be embedded in SQL.
Output: Modified QueryService with MetadataService injection, new _validate_identifier method, wired get_sample_data tool, and comprehensive tests.
</objective>
