```python
class TestSharedMetadataBehavior:
    """Dialect-parametrized shared-behavior tests for MetadataService.

    Uses the `dialect` fixture (MagicMock Engine/Connection/Inspector + real
    DialectStrategy) for tests asserting specific SQL/call shapes, and
    `dialect_inspector` (real SQLAlchemy Inspector + in-memory SQLite for
    generic) for tests asserting real Inspector return shapes.

    Added per Phase 13 / D-08 / D-17 (parallel-add strategy for test_metadata.py).
    Existing `TestListSchemas` and other classes using the `test_engine` fixture
    are preserved for regression baselining and non-parametrized coverage.
    """

    def test_list_schemas_returns_schema_objects(self, dialect_inspector):
        """Shared: list_schemas returns Schema objects regardless of dialect.

        Uses dialect_inspector so the generic case runs against real SQLite
        metadata; mssql/databricks run against the MagicMock inspector and
        configure canned responses inline."""
        if dialect_inspector.name == "generic":
            service = MetadataService(dialect_inspector.engine, dialect=dialect_inspector.dialect)
            schemas = service.list_schemas()
            # SQLite reports 'main' schema by default; assert shape not specific names
            assert all(hasattr(s, "name") for s in schemas)
        else:
            # MagicMock path: configure inspector responses per dialect, then call
            dialect_inspector.inspector.get_schema_names.return_value = ["dbo", "sales"]
            # ... construct MetadataService with mocked engine/dialect/inspector,
            # assert returned Schema objects have .name in {"dbo", "sales"}
            ...

    def test_list_tables_returns_table_objects(self, dialect_inspector):
        """Shared: list_tables returns Table objects for a given schema."""
        ...

    def test_get_table_schema_returns_table_schema_object(self, dialect_inspector):
        """Shared: get_table_schema returns TableSchema with columns.

        Note per META-04 / Phase 11: get_table_schema omits indexes when
        dialect.supports_indexes is False. Assert index-section presence
        reflects dialect_inspector.dialect.supports_indexes."""
        ...
```

Implementation notes:

1. For each test in the new class, prefer `dialect_inspector` when the test cares about real Inspector return shapes; use `dialect` (default MagicMock) when asserting on specific SQL-text or call counts.

2. Honor `dialect.dialect.supports_indexes`:
   - `[mssql]` and `[generic]`: True → expect index section populated
   - `[databricks]`: False → expect index section omitted (per META-04)
   Rather than skipping databricks, assert the conditional behavior.

3. For truly dialect-specific assertions (e.g., MSSQL-optimized DMV queries for list_tables), use `@pytest.mark.dialects('mssql')` rather than branching inside the test body.

4. Do NOT use `@pytest.mark.parametrize` at the class level — parameterization comes from the `dialect`/`dialect_inspector` fixture (indirect). `@pytest.mark.parametrize` is fine for per-test sub-cases within a dialect-parametrized test (e.g., iterating over type samples).

5. Reuse existing helpers (`_row(...)` from Plan 02 files if promoted, or inline MagicMock setup) for MagicMock connection.execute responses. If the existing file has its own helpers, reuse them before inventing new ones.

6. Do NOT modify `test_engine`, `TestListSchemas`, or any existing class in this file in this task. Retirement happens in Task 2 after parity is confirmed.
