# Phase 13.1 Plan 02: WIRING-02 get_sample_data dialect threading — Summary

**One-liner:** Thread `conn_manager.get_dialect(connection_id)` into both `MetadataService` and `QueryService` inside `get_sample_data._sync_work`, eliminating silent sqlglot-dialect degradation to `tsql` with empty `safe_procedures` on Databricks/generic connections.

## What Shipped

3-line source fix + 1 TDD regression test. Before: `MetadataService(engine)` and `QueryService(engine, metadata_service=metadata_svc)` relied on `QueryService`'s auto-infer, which for non-SQLAlchemy-registered dialects (Databricks, generic) fell back to `dialect=None` → `"tsql"` + empty `safe_procedures` inside `validate_query`. After: both services receive the connection's registered `DialectStrategy` explicitly, restoring VALID-01/VALID-02/META-05 at this boundary.
