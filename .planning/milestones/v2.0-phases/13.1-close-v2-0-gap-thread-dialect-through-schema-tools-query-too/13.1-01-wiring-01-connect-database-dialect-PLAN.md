<objective>
Close WIRING-01: `connect_database` in `src/mcp_server/schema_tools.py` builds a one-shot `MetadataService(engine)` without passing the already-resolved dialect. For generic SQLAlchemy URLs, `engine.dialect.name` (e.g., "postgresql") does not match the "generic" registry key, so auto-infer yields `_dialect=None` and `schema_count` in the response is degraded. Fix: thread `dialect=dialect` into that construction. Add two regression tests: a construction assertion and a behavior test that asserts `schema_count` is correct for a generic URL connection.

Purpose: Restore META-01 integration-boundary correctness for `connect_database`'s success response.
Output: One-line source fix + 2 new regression tests.
