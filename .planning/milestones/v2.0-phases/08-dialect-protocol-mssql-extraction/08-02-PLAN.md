    ```python
    class MssqlDialect:
        """SQL Server dialect implementation.

        Encapsulates MSSQL-specific behavior: ODBC connection strings,
        Azure AD authentication, DMV-based fast metadata, bracket quoting.
        """
    ```

    Properties (per D-01, D-02, D-03):
    - `name` -> `"mssql"`
    - `sqlglot_dialect` -> `"tsql"`
    - `supports_indexes` -> `True`
    - `has_fast_row_counts` -> `True`

    Methods:

    **`quote_identifier(self, identifier: str) -> str`** (META-05):
    - Returns `f"[{identifier}]"` -- standard SQL Server bracket quoting
    - Docstring: "Quote using SQL Server square brackets."

    **`create_engine(self, **kwargs) -> Engine`**:
    - Extract logic from `ConnectionManager._build_odbc_connection_string()` and
      `ConnectionManager._create_engine()` into this method.
    - Expected kwargs: `server: str`, `database: str`, `port: int`, `username: str | None`,
      `password: str | None`, `authentication_method: AuthenticationMethod`,
      `trust_server_cert: bool`, `connection_timeout: int`, `query_timeout: int`,
      `pool_config: PoolConfig`, `tenant_id: str | None`, `connection_id: str | None`,
      `disconnect_callback: Callable[[str], None] | None`
    - Internally calls private `_build_odbc_connection_string()` (moved from ConnectionManager)
    - Builds `mssql+pyodbc://` URL with QueuePool configuration from pool_config
    - Handles Azure AD integrated auth via AzureTokenProvider with creator function
    - Sets auth-aware `pool_recycle` for Azure AD connections
    - Attaches query timeout via `@event.listens_for(engine, "connect")`
    - Import from `src.db.dialects.azure_auth import AzureTokenProvider, SQL_COPT_SS_ACCESS_TOKEN`
    - Import PoolConfig from `src.db.connection` (PoolConfig is dialect-agnostic, stays there)

    **`fast_row_counts(self, engine: Engine, schema_name: str | None = None) -> dict[str, int]`**:
    - Execute DMV query against `sys.dm_db_partition_stats` joined with `sys.objects` and `sys.schemas`
    - SQL: `SELECT s.name + '.' + o.name AS table_key, SUM(CASE WHEN ps.index_id IN (0,1) THEN ps.row_count ELSE 0 END) AS row_count FROM sys.dm_db_partition_stats ps JOIN sys.objects o ON ps.object_id = o.object_id JOIN sys.schemas s ON o.schema_id = s.schema_id WHERE o.type = 'U'` with optional `AND s.name = :schema_name` filter
    - GROUP BY `s.name, o.name`
    - Returns `dict[str, int]` mapping `"schema.table"` to row count

    Also move `_classify_db_error()` from connection.py as a module-level function in mssql.py
    (it contains MSSQL-specific SQLSTATE codes and Azure AD token message detection).
    Keep a re-export or reference in connection.py if it's still called there.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MssqlDialect.create_engine | Processes connection parameters (server, credentials) to build ODBC strings |
| MssqlDialect.quote_identifier | Processes identifier strings into SQL fragments |
| MssqlDialect.fast_row_counts | Executes SQL against sys.* DMVs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-02 | Tampering | quote_identifier | mitigate | Bracket quoting is the standard SQL Server approach. Identifiers already validated upstream by _sanitize_identifier regex (`^[a-zA-Z0-9_\s]+$`). quote_identifier adds brackets; it does not replace upstream validation. |
| T-08-03 | Information Disclosure | create_engine | mitigate | Azure AD tokens never logged -- CredentialFilter remains active. No new log statements expose credentials. AzureTokenProvider unchanged (just relocated). |
| T-08-04 | Elevation of Privilege | fast_row_counts | accept | Executes read-only DMV query against sys.dm_db_partition_stats. No user-supplied SQL interpolated -- schema_name passed as SQLAlchemy bound parameter. |
</threat_model>
