<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SQL query text -> validate_query | Untrusted SQL from LLM agents parsed and validated |
| dialect parameter -> sqlglot.parse | Dialect string controls parser behavior |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-04 | Tampering | dialect parameter to validate_query | mitigate | Callers get dialect from DialectStrategy.sqlglot_dialect (protocol-controlled); validate_query does not validate the string itself but sqlglot.parse raises on truly invalid dialect names |
| T-09-05 | Elevation of Privilege | safe_procedures bypass | mitigate | safe_procedures is composed at the QueryService level from dialect.safe_procedures + config allowlist; validate_query cannot expand the set on its own |
| T-09-06 | Tampering | EXEC in non-MSSQL dialect | accept | EXEC parses as garbage (Alias) in non-MSSQL dialects, caught by _GARBAGE_PARSE_TYPES detection — correct denial, just different category (PARSE_FAILURE vs STORED_PROCEDURE) |
</threat_model>
