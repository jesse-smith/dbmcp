## Cross-Phase Integration

**Integration checker result (re-audit, all 3 phases): 15/15 connections wired, 0 orphaned, 0 broken flows, 7/7 E2E flows complete.**

### Critical Seam: Phase 14 → 15 → 15.1 Catalog Chain — SOUND

The catalog-known invariant flows cleanly across all three phases:

1. `DatabricksDialect.create_engine` raises `ValueError("Databricks catalog is required")` on empty/None catalog (databricks.py).
2. Both connect paths route it through `_require_databricks_catalog` → `ConnectionError` with `SHOW CATALOGS` listing (connection.py).
3. `MetadataService._engine_catalog()` reads `engine.url.query["catalog"]` — loud `KeyError` on violation, no silent default; `list_schemas` uses `catalog or self._engine_catalog()` (no fallthrough — IDENT-02).
4. `resolve_identifier` returns `catalog=None` (default path) or `catalog=<requested>` (cross-catalog), gating the param on MSSQL/generic (Phase 15).
5. `cross_catalog = bool(resolved.catalog) and dialect.name == "databricks"` guard appears identically in all three analysis tools; `catalog=resolved.catalog if cross_catalog else None` threads into PKDiscovery / FKCandidateSearch / ColumnStatsCollector (Phase 15.1).
6. All cross-catalog raw SQL uses `CatalogAwareReflector` with per-segment `dialect.quote_identifier` (backtick-escape-safe per CR-01 fix). **No `USE CATALOG` emitted** anywhere in the five modified source files (grep: 0 occurrences).

### Resolver Wiring — CONSISTENT across all 7 namespace-aware tools

`resolve_identifier`: get_table_schema, get_sample_data, get_column_info,
find_pk_candidates, find_fk_candidates. `_assert_catalog_allowed`: list_schemas,
list_tables. `connect_database`/`execute_query` correctly excluded (no namespace params).

### Integration WARNING (non-blocking, carried from prior audit)

`connect_with_url` with a structurally invalid Databricks URL (missing host) re-raises
the raw `_kwargs_from_url` ValueError instead of the enriched catalog error. Only the
error-message quality degrades on an unusual input; the catalog-missing path (the
IDENT-01 target scenario) works correctly. Tracked as tech debt, not a gap.

### New Issues Introduced by Phase 15.1

None. The implementation is purely additive (new `CatalogAwareReflector` /
`quote_tsql_identifier`, new `catalog` constructor params, new `_cross_catalog`
branches). Default-catalog and MSSQL/generic gate paths are structurally unchanged.
