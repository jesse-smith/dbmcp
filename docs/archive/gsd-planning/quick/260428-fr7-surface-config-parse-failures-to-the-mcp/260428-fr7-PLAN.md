<objective>
Preserve TOML parse failures from `load_config()` and surface them through `connect_database` so MCP clients see the real cause instead of the misleading "Available: none" message.

Purpose: Debugging config issues currently requires reading server logs (invisible to MCP clients). This fix makes parse errors discoverable from the client side, preventing the misdiagnosis loop that previously led to mistakenly blaming config *discovery*.

Output: `AppConfig.load_error` field populated on parse failure, `connect_database` branches on it, tests covering both behaviors.
</objective>
