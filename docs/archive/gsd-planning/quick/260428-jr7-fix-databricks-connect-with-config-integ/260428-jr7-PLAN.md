<objective>
Fix two bugs in `ConnectionManager.connect_with_config`'s Databricks branch
(src/db/connection.py:424-432) that together make `dbmcp.toml`-configured
Databricks connections unusable:

  **Bug A:** Only `token` is passed through `resolve_env_vars`. Fields `host`,
  `http_path`, `catalog`, and `schema_name` are used verbatim, so
  `host = "${DATABRICKS_HOST}"` in TOML stays literal.

  **Bug B:** The branch builds a `databricks://...` URL and delegates to
  `connect_with_url`, which calls `dialect.create_engine(sqlalchemy_url=...)`.
  But `DatabricksDialect.create_engine` (src/db/dialects/databricks.py:65-107)
  reads `kwargs["host"]`/`kwargs["http_path"]` and has no `sqlalchemy_url`
  handling, so it always raises `ValueError: Missing required parameter: host`.

Purpose: Make `connect_database(connection_name="databricks-test")` actually
work when the TOML config uses env-var interpolation, and prevent silent
signature drift between `connect_with_config` and `DatabricksDialect`.

Output: Corrected Databricks branch that calls `dialect.create_engine`
directly with resolved kwargs, a small shared helper for post-engine
bookkeeping so `connect_with_url` and the new path can't drift, and a
regression test that fails pre-fix and passes post-fix.
</objective>
