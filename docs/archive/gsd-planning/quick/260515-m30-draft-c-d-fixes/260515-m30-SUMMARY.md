## Out-of-scope items observed

- **Defect A** — `src/config.py:77` and `:243` `"main"` defaults bypass IDENT-01. Independent of D. Recommended next quick task.
- **URL-mode env-var resolution gap** — `${VAR}` placeholders in `sqlalchemy_url` strings are passed through literally; named-config goes through `resolve_env_vars` (`src/db/connection.py:572-576`), URL-mode does not. Probe 3 of the live UAT exposed this: with `?catalog=bmtct` in the URL, the fix correctly forced a fresh pool entry, and the fresh engine attempt then failed auth because the literal `${DATABRICKS_TOKEN}` reached the driver. Pre-fix this was masked because URL probes silently free-rode on the named-config engine's already-authenticated state via the colliding pool key. Now that the pool key correctly disambiguates, the asymmetry is visible. Worth a separate quick task — likely a `resolve_env_vars` call against `parsed.password` (and possibly `parsed.username`/`parsed.host`) inside `connect_with_url` before the dialect builds the engine. Affected file: `src/db/connection.py` (`connect_with_url`, around the `make_url` call). Not a Phase 14 invariant violation; URL-mode env-var resolution was never specified.
- **MEMORY.md mcp_servers note** — already created during prior debug. Documents `dbmcp` (frozen) vs `dbmcp-test` (dev) so probes target the right server.

## Commit plan

Atomic single commit (code + tests together, since they belong to the same red→green increment):

```
fix(connection): include URL query params in pool key (Phase 14 D, C)

_generate_url_connection_id previously hashed only
{backend}://{host}:{port}/{database}, ignoring URL query params. For
Databricks the database URL path is empty and ?catalog= lives in the
query, so all catalogs on one host collapsed to one pool entry —
the pool-reuse short-circuit then returned the wrong engine (Defect D)
and silently bound the user to whichever catalog was cached first
(Defect C). Fold sorted query params into the hashed key. Sorting
keeps ?a=1&b=2 and ?b=2&a=1 stable. Credentials remain excluded
(SQLAlchemy URL.query never contains them).

Note: connection_ids change for any URL with query params. Internal-
only; no documented external dependency on stability.
```

Docs commit (PLAN.md + SUMMARY.md + STATE.md): handled by orchestrator after code commit.
