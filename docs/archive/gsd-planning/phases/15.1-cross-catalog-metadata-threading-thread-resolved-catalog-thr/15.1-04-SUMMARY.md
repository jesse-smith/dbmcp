# Phase 15.1 Plan 04: Catalog-Aware ColumnStatsCollector Summary

Threaded `resolved.catalog` into `ColumnStatsCollector` so that `get_column_info` column statistics target the requested Databricks catalog on BOTH the transpiled aggregate-SQL path and the native-backtick DESCRIBE EXTENDED fast path (Pitfall 5), plus the existence/pattern/type reads — eliminating CR-02 for this path while leaving the default-catalog behavior byte-for-byte unchanged (IDENT-08).

## What Was Built

- **`ColumnStatsCollector.__init__`** gained `catalog: str | None = None`. When set, `_qualified_table` becomes the 3-part TSQL bracket name `[catalog].[schema].[table]` (transpiled to 3-part backticks by `transpile_query`, fixing every aggregate site: basic/numeric/datetime/string). When `None`, the existing 2-part name is preserved.
- **`_is_cross_catalog_databricks` predicate** + **`_reflect_catalog_columns()` helper**: a single gate that routes the four catalog-blind reads (`column_exists`, `get_columns_by_pattern`, `get_column_data_type`, `_resolve_columns_to_analyze`'s all-columns branch) to `CatalogAwareReflector.reflect_columns` over the live `self.connection` on the cross-catalog Databricks branch.
- **`_try_describe_extended_stats` (Pitfall 5)**: builds a 3-part native-backtick name `qi(catalog).qi(schema).qi(table)` when a catalog is set (NOT transpiled — mirrors `get_sample_data` in `src/db/query.py`), else the existing 2-part name. Every segment quoted via `dialect.quote_identifier`.
- **`tests/unit/test_column_stats.py`**: new `TestCrossCatalogColumnStats` class with a catalog-discriminating fake connection that returns populated rows only when the SQL carries the resolved-catalog 3-part backtick name, asserts no `USE CATALOG` on every execute, and verifies both paths plus the default-path guards.

## Threat Mitigations Applied

- **T-15.1-09 (SQLi, DESCRIBE EXTENDED fast path):** every segment of the 3-part fast-path name is quoted via `dialect.quote_identifier` (no raw concatenation of user input). `test_describe_extended_fast_path_three_part` asserts the quoted 3-part form is emitted.
- **T-15.1-10 (SQLi, aggregate SQL):** the 3-part `_qualified_table` is TSQL brackets transpiled by sqlglot; segments are identifiers, not concatenated raw user SQL. `test_qualified_table_three_part` verifies the bracket form transpiles to backticks.
- **T-15.1-11 (session-catalog tampering):** neither path emits `USE CATALOG`; both use stateless fully-qualified names. The discriminating fake asserts `USE CATALOG` absence on every execute (`test_no_use_catalog`), and the source grep gate confirms 0 occurrences.
