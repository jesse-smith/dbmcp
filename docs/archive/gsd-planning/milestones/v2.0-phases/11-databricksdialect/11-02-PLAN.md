    ```python
    def _parse_databricks_table_properties(
        self, table_name: str, schema_name: str, catalog: str
    ) -> dict:
    ```

    Implementation (per research code example):
    - Build qualified name using dialect quoting: `` `{catalog}`.`{schema_name}`.`{table_name}` ``
    - Execute `DESCRIBE TABLE EXTENDED {qualified_name}` via `self.engine.connect()`
    - Parse output rows (col_name, data_type columns):
      - Track section state: `in_partition_section`, `in_detail_section`
      - Section markers: `"# Partition Information"` starts partition section, `"# Detailed Table Information"` starts detail section, `"#"` prefix = skip header rows, blank row resets partition section
      - In partition section: collect non-header col_name values into partition_cols list
      - In detail section: map known keys to output fields:
        - "Owner" -> "owner"
        - "Provider" -> "storage_format"
        - "Type" -> "table_type_detail"
        - "Created Time" -> "created_time"
        - "Location" -> "location"
    - If partition_cols is non-empty, add `"partition_columns": partition_cols`
    - Return the dict (may be empty if no detail section found)
    - Catch ALL exceptions (SQLAlchemyError and general Exception per research pitfall 1) and return empty dict with warning log -- never let DTE parsing failure break get_table_schema

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP tool params -> SQL | catalog, schema_name, table_name from LLM agent used in SHOW/DESCRIBE queries |
| DTE output -> response | Parsed text from Databricks may contain unexpected content |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-04 | Tampering | DESCRIBE TABLE EXTENDED query | mitigate | All identifiers (catalog, schema, table) backtick-quoted via dialect.quote_identifier() before embedding in SQL. No string interpolation of unquoted user input. |
| T-11-05 | Tampering | SHOW SCHEMAS IN / SHOW TABLES IN queries | mitigate | Catalog and schema names backtick-quoted via dialect.quote_identifier(). Prevents SQL injection via catalog parameter. |
| T-11-06 | Denial of Service | _parse_databricks_table_properties | mitigate | Wrapped in try/except with warning log. DTE parsing failure returns empty dict, never breaks get_table_schema. Timeout inherited from engine's query_timeout. |
| T-11-07 | Information Disclosure | DTE output parsing | accept | DTE output may contain internal paths (Location field). This is metadata the user is authorized to see via their Databricks token. No additional filtering needed. |
</threat_model>
