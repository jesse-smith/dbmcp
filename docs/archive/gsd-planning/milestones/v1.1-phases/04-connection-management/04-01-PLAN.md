<objective>
Add auth-aware pool recycling so Azure AD connections are recycled before token expiry, and handle token re-acquisition failures gracefully.

Purpose: CONN-01 requires that pooled connections with expired Azure AD tokens are discarded before use. The existing creator callable + pool_pre_ping already handles token refresh on new connections; this plan sets pool_recycle=2700 for Azure AD to ensure connections are proactively recycled before the ~3600s token lifetime expires.

Output: Modified PoolConfig with azure_ad_pool_recycle field, auth-aware _create_engine, tests proving the behavior.
</objective>

<task type="auto" tdd="true">
  <name>Task 2: Handle token re-acquisition failure with auto-disconnect</name>
  <files>src/db/connection.py, tests/unit/test_connection.py</files>
  <behavior>
    - Test: When creator callable's get_token() raises ConnectionError, the connection_id is auto-disconnected from ConnectionManager
    - Test: When creator callable's get_token() raises ConnectionError, the original error is re-raised (not swallowed)
    - Test: Auto-disconnect logs at DEBUG level
    - Test: If connection_id is not in engines (edge case), auto-disconnect is a no-op (no crash)
  </behavior>
  <action>
1. The challenge: the `creator` callable inside `_create_engine` doesn't have access to `connection_id` (it's not yet stored). The creator is called by SQLAlchemy's pool when it needs a new connection -- this can happen at initial connect time or later during pool refresh.

Approach: Store a reference to the engine + connection_id after creation so the creator can trigger cleanup. Specifically:
- Wrap the existing Azure AD `creator()` function to catch `ConnectionError` from `get_token()`. When caught:
  - Log at DEBUG: `f"Azure AD token re-acquisition failed, cleaning up connection"`
  - Call `self.disconnect(connection_id)` if `connection_id` is in `self._engines` (it won't be during initial connect, only during pool refresh)
  - Re-raise the ConnectionError

This requires the creator to capture `self` and `connection_id` via closure. But `connection_id` is computed before `_create_engine` is called (in `connect()`), so pass it as a parameter to `_create_engine`.

2. Update `_create_engine` signature to accept `connection_id: str | None = None`.

3. In the `creator()` closure for Azure AD Integrated, wrap the `provider.get_token()` call:
```python
def creator():
    try:
        token = provider.get_token()
    except ConnectionError:
        logger.debug("Azure AD token re-acquisition failed, cleaning up connection")
        if connection_id and connection_id in self._engines:
            self.disconnect(connection_id)
        raise
    packed = provider.pack_token_for_pyodbc(token)
    return pyodbc.connect(odbc_conn_str, attrs_before={SQL_COPT_SS_ACCESS_TOKEN: packed})
```

4. Update the `connect()` method's call to `_create_engine` to pass `connection_id`.

5. Write TDD tests: mock `AzureTokenProvider.get_token` to raise `ConnectionError`, verify that `disconnect()` is called and the error propagates. Use a ConnectionManager with a pre-populated `_engines` dict to simulate the pool-refresh scenario.
