  <files>src/config.py, tests/unit/test_config.py</files>
  <behavior>
    - load_config() with no file on disk -> AppConfig with all defaults
    - load_config() with valid dbmcp.toml -> parsed config with named connections and defaults
    - load_config() with malformed TOML -> logs warning, returns AppConfig defaults
    - Local dbmcp.toml takes precedence over ~/.dbmcp/config.toml
    - DefaultsConfig validates bounds: query_timeout 5-300, text_truncation_limit 100-10000, sample_size 1-1000, row_limit 1-10000
    - Out-of-range values: log warning, use hardcoded default for that field
    - ConnectionConfig stores server, database, port, auth_method, username, password, trust_server_cert, connection_timeout, tenant_id
    - resolve_env_vars("${VAR_NAME}") with VAR_NAME set -> resolved value
    - resolve_env_vars("${MISSING}") -> raises ValueError with clear message
    - resolve_env_vars("plain string") -> unchanged
    - SP allowlist: valid identifier names accepted, invalid (containing SQL injection chars) rejected with warning
    - SP allowlist: schema-qualified names like "dbo.my_proc" accepted
    - init_config() stores singleton; get_config() returns it; get_config() before init returns defaults
  </behavior>
  <action>
    Write tests first in tests/unit/test_config.py. Use tmp_path fixtures to create temporary TOML files. Use monkeypatch for env vars and path overrides. Cover every behavior listed above.

    Then create src/config.py with:
    - `DefaultsConfig` frozen dataclass: query_timeout=30, text_truncation_limit=1000, sample_size=5, row_limit=1000
    - `ConnectionConfig` frozen dataclass: server, database, port=1433, authentication_method="sql", username=None, password=None, trust_server_cert=False, connection_timeout=30, tenant_id=None
    - `AppConfig` frozen dataclass: defaults (DefaultsConfig), connections (dict[str, ConnectionConfig]), allowed_stored_procedures (frozenset[str])
    - `_ENV_VAR_PATTERN = re.compile(r'\$\{([^}]+)\}')` for ${VAR} syntax
    - `resolve_env_vars(value: str) -> str` — replaces all ${VAR} occurrences, raises ValueError for unresolved vars
    - `_find_config_file() -> Path | None` — checks ./dbmcp.toml then ~/.dbmcp/config.toml
    - `load_config() -> AppConfig` — find file, parse TOML, validate, return frozen config. On error: log warning via get_logger(__name__), return AppConfig()
    - `_parse_config(raw: dict) -> AppConfig` — processes raw TOML dict into dataclasses, validates bounds, validates SP name identifiers with `^[a-zA-Z_][\w.]*$`
    - `_validate_defaults(raw_defaults: dict) -> DefaultsConfig` — checks min/max bounds per field, warns and uses default on violation
    - Module singleton: `_config: AppConfig | None = None`, `init_config() -> AppConfig`, `get_config() -> AppConfig`

    Important: env vars are NOT resolved during load_config(). They stay as "${VAR}" strings in ConnectionConfig. resolve_env_vars() is called at connection time by schema_tools.py.
