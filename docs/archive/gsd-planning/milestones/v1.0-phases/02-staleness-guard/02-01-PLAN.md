```python
def extract_fields(docstring: str) -> dict:
    """Extract field declarations from a TOON docstring Returns section.

    Returns dict with:
      "top_level": set of field names at top level
      "nested": dict mapping parent field -> set of child field names
      "conditional": dict mapping field name -> condition string (e.g., "on error only")
    """
```

Run tests -- they must all FAIL (module doesn't exist yet).

GREEN: Create `tests/staleness/__init__.py` (empty) and `tests/staleness/docstring_parser.py`. Implement `extract_fields()` using:
- `inspect.cleandoc()` to normalize indentation
- Regex to find "Returns:" section and parse `field_name: type` lines
- Indentation tracking to detect nesting (base indent = top-level, deeper = nested under previous list/dict field)
- Parse `// condition` annotations from field lines
- Stop parsing at next section header or end of docstring

Key implementation details:
- Use `re.compile(r'^(\s+)(\w+):\s+(.+?)(?:\s+//\s*(.+))?$')` to match field lines
- The Returns section starts after a line matching `Returns:` (strip whitespace)
- Skip the description line ("TOON-encoded string with...")
- Base indent is the indent of the first field line encountered
- A field at base indent is top-level
- A field with indent > base indent is nested under the most recent list/dict parent
- Annotations after `//` go into the `conditional` dict

Run tests -- they must all PASS.
